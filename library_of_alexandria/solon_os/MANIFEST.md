## Doctrine file placement rule (enforced by `bin/alexandria-preflight.sh`)

New doctrine files MUST land in one of:
- `plans/` (DRs, blueprints, control-loop packages)
- `plans/<sub-project>/` (merchant-stack-applications, control-loop, briefs)
- `baselines/` (perf + capacity baselines)
- `templates/` (proposal templates, brand voice files)
- `library_of_alexandria/<section>/` (catalog notes, manifests)
- VPS `/opt/amg-titan/solon-corpus/` (harvested raw corpus)
- Repo root ONLY for top-level operating docs (CORE_CONTRACT, CLAUDE, INVENTORY, RADAR, README, RELAUNCH_CLAUDE_CODE, SESSION_PROMPT, ALEXANDRIA_INDEX)

Anything outside those paths triggers a warning at commit/build time.
